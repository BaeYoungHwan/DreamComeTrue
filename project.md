=== 프로젝트 초기화 양식 ===
1. 프로젝트명:
local-farm-inventory (로컬 팜 인벤토리)

2. 한 줄 설명 (무엇을 위해 만드는가):
1000평 규모 다품종 소량 생산 농가를 위한 로컬 출하용 재고 관리 및 매출 정산 시스템

3. 기술 스택:
Python 3.12 + Streamlit (빠른 UI 구현용) + SQLite (가벼운 로컬 DB)
(※ 아직 미정이시라면, 혼자 빠르게 만들고 VSCode CLI로 제어하기 가장 쉬운 조합으로 임시 제안해 드립니다.)

4. 배포 환경:
농장 내 PC에서 로컬 실행 (온프레미스) 또는 Streamlit Community Cloud (무료 배포)

5. 타겟 유저 / 페르소나:

주 사용자: 농장주 본인 및 가족 (직접 수확 및 출하를 기록하는 사람)

6. 성공 지표 (KPI):

수기 장부 기록 및 엑셀 정산 시간 70% 단축

품목별 당일 재고 및 당월 누적 매출 실시간 확인 가능

7. MVP 핵심 기능 (반드시 구현할 것):

품목 등록 및 관리: 다품종 관리를 위한 농작물 이름, 규격(kg, 박스 등) 등록

입출고(재고) 관리: 수확량 입력(재고 증가), 로컬 출하량 입력(재고 감소)

판매 및 매출 관리: 출하처별(로컬푸드 매장, 직거래 등) 판매 수량 및 단가 입력

간이 대시보드: 오늘 출하된 농작물 현황, 이달의 누적 판매 금액 및 품목별 판매 순위 시각화(그래프)

8. MVP 제외 사항 (이번 버전에 포함하지 않을 것):

복잡한 회원가입 및 사용자 권한 관리 (단일 사용자용으로 개발)

바코드/QR코드 스캔을 통한 재고 인식 기능

고도화된 세무/회계 증빙 서류 자동 발급 기능

외부 쇼핑몰 연동 및 실시간 알림 톡 기능

9. 비기능 요건 (성능 / 보안 / 가용성 목표):

성능: 로컬 가동이므로 데이터 조회 및 입력 반응 속도 1초 미만

보안: 외부 노출이 없는 로컬 DB 사용, 데이터 유실 방지를 위한 자동 SQLite 파일 백업 기능

가용성: 개인 사용 목적이므로 별도의 SLA(가동률) 타겟 없음

10. 팀 구성 / 일정:

팀: 개인 프로젝트 (1인 개발)

MVP 마감: 미정

런칭 목표: 미정

11. 제약사항 / 특이사항:

인터넷 연결이 불안정한 농장 환경에서도 구동 가능해야 함 (로컬 우선)

외부 유료 API 의존성 전면 배제 (비용 $0 목표)

12. 이 프로젝트에서 Claude에게 특별히 지켜야 할 것:

코드 수정 시 기존 DB 스키마(데이터 구조)가 깨지지 않도록 마이그레이션 코드를 함께 제공할 것

복잡한 디자인 프레임워크 대신 직관적이고 텍스트가 큼직한 UI 구성을 우선할 것
