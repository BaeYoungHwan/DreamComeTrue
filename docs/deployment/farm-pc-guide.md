# 농장 PC 설치·운영 가이드

> 로컬 팜 인벤토리는 **농장 PC에서 로컬로 실행**합니다 (외부 URL 접속 아님).
> 이유: 농장 인터넷이 불안정해 상시 외부 접속에 의존하면 오히려 불편하고,
> 실제 장부 데이터를 외부 서버에 맡기지 않아 데이터가 안전합니다.

---

## 1. 최초 설치 (한 번만)

### 1) Python 설치
[python.org](https://www.python.org/downloads/)에서 Python 3.12 다운로드 후 설치.
**설치 화면에서 "Add python.exe to PATH" 체크박스를 반드시 체크**하세요.

### 2) Git 설치
[git-scm.com](https://git-scm.com/downloads)에서 다운로드 후 기본 설정으로 설치.

### 3) 저장소 받기
바탕화면이나 문서 폴더에서 마우스 우클릭 → "Git Bash Here" (또는 명령 프롬프트) 실행 후:

```
git clone https://github.com/BaeYoungHwan/DreamComeTrue.git 로컬팜인벤토리
```

`로컬팜인벤토리` 폴더가 생성됩니다.

### 4) 필요한 프로그램 설치
`로컬팜인벤토리` 폴더 안의 **`앱_업데이트.bat`**을 더블클릭하세요.
(최초 실행 시 "이미 최신 상태입니다" 메시지가 뜨는 게 정상입니다 — 이어서 필요한 패키지를 설치합니다.)

### 5) 실행
같은 폴더의 **`앱_실행.bat`**을 더블클릭하면 브라우저가 자동으로 열립니다.

### 6) 바탕화면 바로가기 (선택)
`앱_실행.bat` 우클릭 → 보내기 → 바탕화면에 바로가기 만들기.
이후에는 바탕화면 아이콘만 더블클릭하면 됩니다.

---

## 2. 일상 사용

- **실행**: `앱_실행.bat` 더블클릭 → 자동으로 브라우저가 열림
- **종료**: 검은 창(명령 프롬프트)을 닫으면 프로그램이 종료됨
- 인터넷 연결 없이도 정상 동작합니다 (실행 파일이 이미 설치되어 있다면).

---

## 3. 데이터는 어디에 있나요?

- 실제 데이터: `로컬팜인벤토리\data\farm.db` (이 PC에만 있음, 외부로 전송되지 않음)
- 자동 백업: 앱을 실행할 때마다 `로컬팜인벤토리\backups\` 폴더에 시점별 복사본이 쌓입니다.
- **권장**: 가끔 `backups` 폴더를 통째로 USB에 복사해두면 PC 고장 시에도 안전합니다.

---

## 4. 유지보수 흐름 (문제 발견 시)

농장 PC에서 쓰다가 불편한 점이나 고쳐야 할 부분을 발견하면:

1. 무엇이 불편한지 개발자(집 PC 사용자)에게 알려준다.
2. 개발자가 집 PC에서 수정하고 GitHub에 올린다(`git push`).
3. 농장 PC에서 **`앱_업데이트.bat`**을 더블클릭한다 (인터넷 연결 필요, 이때만).
4. 업데이트 완료 후 **`앱_실행.bat`**으로 다시 실행한다.

업데이트는 이 4단계만 반복하면 됩니다. 평소 사용에는 인터넷이 필요 없습니다.

---

## 5. 문제가 생기면

- `앱_실행.bat` 창에 빨간 글씨(오류)가 뜨면, 그 내용을 사진 찍어 개발자에게 전달해주세요.
- 프로그램이 아예 안 켜지면 `앱_업데이트.bat`을 먼저 한 번 실행해보세요.
