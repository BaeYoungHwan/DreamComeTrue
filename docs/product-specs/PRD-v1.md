# local-farm-inventory PRD v1

> 버전: v1 | 작성일: 2026-07-18 | 상태: Draft

## 1. 개요

1000평 규모 다품종 소량 생산 농가를 위한 로컬 출하용 재고 관리 및 매출 정산 시스템.

## 2. 문제 정의

농장주는 현재 수기 장부와 엑셀로 다품종 소량 생산 작물의 수확·출하·매출을 기록하고 있다. 품목이 다양하고 출하처(로컬푸드 매장, 직거래 등)가 여러 곳이라 수기 정산에 시간이 많이 들고, 특정 시점의 재고량이나 누적 매출을 즉시 확인하기 어렵다. 인터넷 연결이 불안정한 농장 환경에서도 안정적으로 동작하는 가벼운 로컬 도구가 필요하다.

## 3. 타겟 유저

| 페르소나 | 설명 | 주요 시나리오 |
|----------|------|---------------|
| 농장주 본인 및 가족 | 직접 수확·출하를 기록하는 단일 사용자(가족 포함). PC 1대에서 순차 입력, 모바일 접근 불필요 | 수확량 입력 → 출하량 입력 → 출하처별 판매 기록 → 대시보드로 당일/당월 현황 확인 |

## 4. 성공 지표 (KPI)

- 수기 장부 기록 및 엑셀 정산 시간 70% 단축
- 품목별 당일 재고 및 당월 누적 매출 실시간 확인 가능

## 4-1. 완료 기준 (Definition of Done)

핵심 시나리오(품목 등록 → 입고 → 출고 → 폐기/손실 반영 → 매출 정산 → 대시보드 확인)가 오류 없이 1회 정상 동작하면 MVP 완성으로 간주한다.

추가 기능(품목별 커스텀 속성, 엑셀 Export, 기간별 정산 리포트)은 각각 오류 없이 1회 동작하면 해당 기능 완성으로 간주한다.

## 4-2. KPI 측정 기준

시간 측정 인프라(스톱워치 기능 등)를 앱에 추가하지 않는다 — 가족 단위 개인 프로젝트 규모에 맞지 않는 과설계다. 대신 아래 방식으로 체감을 확인한다.

- **측정 방법**: 별도 도구 없이 사용 전후를 구두로 비교한다.
  - 사용 전 기준값: 기존 수기 장부 + 엑셀 정산에 걸리던 시간(예: "월말 정산에 반나절")을 농장주 본인 진술로 기록해둔다.
  - 사용 후 확인: 농장 PC 배포 후 **2주 정도 실사용**한 시점에, "장부 정리·정산이 예전보다 얼마나 빨라졌다고 느끼는지"를 농장주(부모님)에게 직접 묻는다.
- **판단 기준**: 정량적 초 단위 측정이 아니라 "확실히 빨라졌다 / 비슷하다 / 더 불편하다" 3단계 체감 응답으로 충분하다. "확실히 빨라졌다"면 KPI 달성으로 간주.
- **함께 확인할 것**: 대시보드에서 "품목별 당일 재고·당월 누적 매출을 즉시 확인할 수 있었는지"도 같은 시점에 함께 질문한다(4번 성공 지표 2번째 항목).
- **기록 위치**: 확인 결과는 이 문서(4-2)에 후속 업데이트로 남기거나, `docs/ref/session-state.md`에 간단히 메모한다.

## 5. 기술 스택

Python 3.12 + Streamlit (최신 안정 버전) + SQLite (Python 내장, 최신)

## 6. 배포 환경

농장 내 PC에서 로컬 실행 (온프레미스) 또는 Streamlit Community Cloud (무료 배포)

## 7. MVP 핵심 기능

- [ ] 품목 등록 및 관리: 농작물 이름, 규격(kg, 박스 등) 등록
- [ ] 입출고(재고) 관리: 수확량 입력(재고 증가), 로컬 출하량 입력(재고 감소), 폐기/손실 수량 입력(재고 감소) — 재고 = 입고 − 출고 − 폐기
- [ ] 판매 및 매출 관리: 출하처별(로컬푸드 매장, 직거래 등) 판매 수량 및 단가 입력 (품목 고정 단가 없음, 출하 시마다 직접 입력)
- [ ] 간이 대시보드: 오늘 출하된 농작물 현황, 이달의 누적 판매 금액 및 품목별 판매 순위 시각화(그래프)
- [ ] 품목별 커스텀 속성 관리: 다품종 특성상 품목마다 서로 다른 속성(예: 당도, 재배방식, 포장단위)을 자유롭게 추가·삭제 (정산 리포트에는 미반영, 품목 관리 화면 전용)
- [ ] 엑셀(.xlsx) Export: 재고·판매 데이터를 개인 기록·백업용 엑셀 파일로 내보내기
- [ ] 기간별 정산 리포트: 시작일~종료일 지정 후 품목별 출하량·판매대금 집계 조회 및 엑셀 출력

## 8. MVP 제외 사항 ⚠️

> Claude는 아래 항목을 임의로 구현하지 않습니다.

- 복잡한 회원가입 및 사용자 권한 관리 (단일 사용자용으로 개발)
- 바코드/QR코드 스캔을 통한 재고 인식 기능
- 고도화된 세무/회계 증빙 서류 자동 발급 기능
- 외부 쇼핑몰 연동 및 실시간 알림 톡 기능
- 한글(.hwp) 형식 Export (엑셀만 우선 구현, 필요 시 다음 버전에서 검토)

## 9. 제약사항

- 외부 유료 API 의존성 전면 배제 (비용 $0 목표)
