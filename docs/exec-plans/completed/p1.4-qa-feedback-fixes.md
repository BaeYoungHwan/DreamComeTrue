# P1.4 — QA 시뮬레이션 피드백 5건 반영

## 배경
사진 속 실제 로컬푸드 정산 장부를 재현한 데모 데이터로 앱을 직접 띄워(`data/qa_demo.db`, 운영 DB 미사용) 사용자가 QA를 진행했고, `시뮬레이션 피드백.txt`에 5건이 기록됨. 사용자에게 두 항목(재고 단위 표시 원인, 커스텀 속성 처리 방향)을 직접 확인 후 반영. 계획 원문: `.claude/plans/fluttering-growing-heron.md`.

## 이슈 및 수정 내역

1. **재고가 "g"으로 이상하게 보임** — 조사 결과 수량과 `item['unit']`을 이어붙이는 4곳 중 3곳(변형재고/입출고내역/판매내역)에 공백이 없어 `"3.96500g"`처럼 붙어 보이는 포맷 버그로 확인. → `app.py`의 해당 3개 f-string에 공백 추가.

2. **리포트가 합산만 되어 상세 리포트 필요** — `period_report()`가 `GROUP BY items.name`으로 품목 단위 합산만 반환, 날짜별 개별 판매 건을 보는 화면이 없었음. → `src/export/report.py`에 `detailed_period_report()`(sales↔items↔item_variants↔channels 조인, 날짜순 개별 건 반환), `detailed_report_to_excel_bytes()` 추가. 정산 리포트 탭에 "상세 리포트 보기" 버튼 + 표 + 엑셀 다운로드 추가(기존 `report_rows`와 동일한 `st.session_state` 패턴).

3. **가격 마스터 소수점 불필요** — 변형 기준단가/판매 단가/실제 통장입금액 `number_input` 3곳이 `min_value=0.0, step=100.0`(float)이라 위젯에 소수점이 표시됨. → 세 곳 모두 `min_value=0, step=100, format="%d"`로 정수화. `_apply_variant_price()` 콜백의 `float()` 캐스팅도 `int()`로 변경.

4. **커스텀 속성 용도 불명** — 실사용에서 당도/재배방식 자유 메모 기능의 목적을 이해하지 못함 → 이번엔 UI 제거로 결정(사용자 확인). `app.py` 품목 상세의 캡션+추가/삭제 위젯 블록을 통째로 제거. `items.py`의 `update_custom_attributes`/`custom_attributes` 컬럼·백필은 스키마 변경 없이 그대로 유지(추후 필요 시 UI만 재부착 가능).

5. **재고를 볼 곳이 없음** — 기존엔 "입출고" 탭 하단에만 품목 합계 재고가 있었고(변형별 분해 없음), 다른 탭에서는 전혀 안 보였음. → `src/inventory/stock.py`에 `stock_overview()`(품목 전체 순회, 변형 있으면 변형별로, 없으면 품목 전체로 한 행) 추가. `app.py`에 새 첫 번째 탭 "재고 현황" 신설(8개 탭 체제), 앱을 열면 가장 먼저 보이는 화면으로 배치.

## 변경 파일
- `src/export/report.py` — `detailed_period_report()`, `detailed_report_to_excel_bytes()`
- `src/inventory/stock.py` — `stock_overview()`
- `app.py` — 탭 구성 8개로 확장("재고 현황" 신설), 공백 버그 3곳, 가격 입력 3곳 정수화, 커스텀 속성 UI 제거, 상세 리포트 UI 추가
- `tests/test_report.py`, `tests/test_stock.py`, `tests/test_app.py` — 신규 테스트 7건, 탭 개수 테스트 갱신, 커스텀 속성 골든패스 테스트 제거(대상 UI 삭제로 인해)
