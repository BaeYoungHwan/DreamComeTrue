# P1.1 — 실사용 시뮬레이션 피드백 5건 수정

## 배경
부모님이 실제 쓸 예정인 로컬 팜 인벤토리 앱을 사용자가 직접 구동해 시뮬레이션한 결과 5가지 문제가 보고됨. 코드를 직접 조사해 원인을 특정하고 수정.

## 이슈 및 수정 내역

1. **필드값 영어 표시** — `app.py`의 `st.table()`이 dashboard/report 함수가 반환하는 영어 dict key(`item_name`, `quantity`, `total_quantity`, `total_amount`)를 그대로 헤더로 노출. → 화면 표시 직전 한글 key(`품목`/`수량`/`출하량`/`판매대금`)로 변환하는 방식으로 수정 (백엔드 함수의 key는 테스트 호환을 위해 유지). 커스텀 속성도 dict repr 대신 `- 키: 값` 목록형으로 변경.

2. **품목 삭제 버튼 무반응** — `stock_transactions`/`sales`가 `items`를 FK로 참조하고 `PRAGMA foreign_keys = ON`이 걸려 있어, 거래 기록이 있는 품목 삭제 시 `sqlite3.IntegrityError`가 미처리 상태로 발생. → `src/inventory/items.py`에 `ItemInUseError` 추가, `delete_item()`이 사전에 거래/판매 건수를 확인해 있으면 친절한 한글 메시지로 차단 (정책은 사용자 확인: "기록 있으면 삭제 차단").

3. **날짜 입력란 부재** — `record_transaction`/`record_sale`에 날짜 파라미터가 없어 항상 `datetime('now')`로만 기록됨. → `occurred_on`/`sold_on` 옵션 파라미터 추가, `app.py` 입출고·판매 탭에 `st.date_input` 추가해 연결.

4. **정산 리포트 엑셀 다운로드 시 화면 소실** — `st.download_button`이 `if st.button("리포트 조회"):` 블록 안에 중첩되어, 다운로드 버튼 클릭이 유발하는 rerun에서 조회 버튼 조건이 다시 False가 되어 표·금액이 사라지는 Streamlit 전형적 버그. → 조회 결과를 `st.session_state`에 저장하도록 리팩터링 (전체 데이터 백업 버튼도 동일 패턴이라 함께 수정).

5. **커스텀 속성 의미 불명** — 설명 문구가 전혀 없었음. → `st.caption`으로 용도 설명 추가, `text_input`에 placeholder(예: 당도, 10 브릭스) 추가.

## 후속 — 개별 입출고·판매 내역 삭제 기능

2번 수정만으로는 "기록이 있으면 삭제 차단"까지만 되고, 그 기록 자체를 지울 방법이 없어 실질적으로 상추를 삭제할 수 없다는 후속 피드백을 받음. 품목 상세(expander)에 개별 내역을 조회·삭제하는 UI를 추가:
- `src/inventory/stock.py`: `list_transactions()` — 판매(sales)에 연결되지 않은 입출고 내역만 반환(판매 연동 출고는 판매 내역 쪽에서 관리), `delete_transaction()`
- `src/sales/sales.py`: `list_sales()`, `delete_sale()` — 판매 삭제 시 연결된 shipment `stock_transactions` 행도 함께 삭제해 재고를 원복
- `app.py`: 품목 expander에 "입출고 내역"/"판매 내역" 섹션과 각 행별 삭제 버튼 추가, 기록이 하나도 없으면 "바로 삭제할 수 있습니다" 안내

## 변경 파일
- `src/inventory/items.py` — `ItemInUseError`, `delete_item` 사전 검증
- `src/inventory/stock.py` — `record_transaction(..., occurred_on=None)`, `list_transactions()`, `delete_transaction()`
- `src/sales/sales.py` — `record_sale(..., sold_on=None)`, `list_sales()`, `delete_sale()`
- `app.py` — 전 항목 UI 반영 + 입출고·판매 내역 삭제 UI
- `tests/test_items.py`, `tests/test_stock.py`, `tests/test_sales.py` — 신규 테스트 7건 추가
