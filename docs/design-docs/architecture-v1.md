## 1. 시스템 개요

농장주 본인 및 가족이 사용하는 1인용 로컬 재고·매출 관리 도구. 다품종 소량 생산 작물의 품목/규격을 등록하고, 수확(입고)과 로컬 출하(출고)를 기록해 재고를 자동 계산하며, 출하처별 판매 기록으로 매출을 정산한다. 오늘의 출하 현황과 이달 누적 매출을 대시보드로 즉시 확인할 수 있다.

│      도메인 레이어 (src/)     │

## 3. 레이어 구조

```
local-farm-inventory/
├── app.py                  # Streamlit 진입점 (페이지 라우팅)
├── src/
│   ├── core/                # DB 연결, 백업 스케줄러, 공통 유틸
│   ├── inventory/            # 품목 등록(커스텀 속성 포함), 입출고(재고) 도메인 로직
│   ├── sales/                 # 판매·매출 정산 도메인 로직
│   ├── dashboard/              # 대시보드 집계·시각화
│   └── export/                  # 기간별 정산 리포트 집계 + 엑셀(.xlsx) 생성
├── data/
│   └── farm.db               # SQLite 파일 (gitignore 대상)
├── backups/                    # 자동 백업 저장소 (gitignore 대상)
└── tests/
```

## 4. 데이터 흐름

**입고(수확) 기록 → 재고 반영**
1. 사용자가 품목/규격/수확량 입력 (Streamlit 폼)
2. `src/inventory`에서 유효성 검증 (품목 존재 여부)
3. SQLite `stock_transactions`에 입고 기록 저장, 현재 재고량 갱신

**출하(판매) 기록 → 재고 차감 + 매출 반영**
1. 사용자가 품목/출하처/수량/단가 입력 (품목 마스터에는 고정 단가 필드 없음 — 판매 레코드마다 단가를 직접 저장)
2. `src/inventory`에서 재고량 검증 (음수 방지 — 출하량이 현재 재고를 초과하면 거부)
3. 재고 차감 + `src/sales`에서 매출 레코드 생성
4. `src/dashboard`가 당일/당월 집계 쿼리로 대시보드 갱신

**폐기/손실 기록 → 재고 차감**
1. 사용자가 품목/규격/폐기 수량 입력
2. `src/inventory`에서 재고량 검증 (음수 방지 동일 적용)
3. `stock_transactions` 테이블에 거래유형(`harvest` / `shipment` / `loss`)으로 구분 저장, 재고 차감
4. 최종 재고 = 입고(harvest) − 출고(shipment) − 폐기(loss)

**품목별 커스텀 속성 관리**
1. 사용자가 품목 관리 화면에서 임의의 속성명(key)·값(value) 쌍을 자유롭게 추가·삭제 (예: 당도, 재배방식, 포장단위)
2. `src/inventory`가 `items.custom_attributes` (JSON TEXT 컬럼)에 저장 — 품목마다 속성 구성이 달라도 스키마 변경 없이 대응
3. 커스텀 속성은 품목 관리 화면 전용이며, 정산 리포트 등 다른 화면에는 반영되지 않음

**기간별 정산 리포트 & 엑셀 Export**
1. 사용자가 시작일~종료일 지정
2. `src/export`가 기간 내 `sales` 레코드를 품목별로 집계 (출하량 합계, 판매대금 합계)
3. Streamlit 화면에 집계 표를 표시하고, 사용자가 요청 시 `src/export`가 openpyxl로 `.xlsx` 파일을 생성해 다운로드 제공

## 5. 주요 설계 결정

| 결정 | 선택 | 이유 | ADR |
|------|------|------|-----|
| 레이어 구조 | UI / 도메인(inventory, sales) / 데이터 3계층 | 소규모 단일 사용자 앱에 맞는 최소 구조 | - |
| 인증 방식 | 없음 | 단일 사용자, 외부 미노출 | - |
| 데이터 저장 | SQLite 단일 파일 | 서버 불필요, 완전 로컬 구동 | - |
| 배포 전략 | 로컬 PC 우선 | 인터넷 불안정 환경 대응 | - |
| 품목 속성 저장 방식 | `items.custom_attributes` (JSON TEXT 컬럼) | 다품종마다 속성이 달라 고정 컬럼으로 대응 불가, 스키마 변경 없이 유연하게 확장 | - |
| Export 형식 | 엑셀(.xlsx)만 우선, 한글(.hwp) 제외 | openpyxl로 구현 용이, 한글 라이브러리 의존성·복잡도가 개인 기록용 목적엔 과함 | - |

## 6. 비기능 요건 달성 전략

- **성능**: SQLite 로컬 파일 접근 + 소량 데이터(1000평 규모, 다품종이라도 레코드 수 적음)로 1초 미만 반응 자연 달성
- **보안**: 외부 네트워크 노출 없음, `src/core`에 주기적 자동 백업 함수 구현 (예: 앱 시작 시 또는 일 1회 `backups/`에 타임스탬프 파일 복사)
- **유지보수성**: `inventory`/`sales` 도메인 규칙에 대한 단위 테스트 우선 작성

## 7. 보안 고려사항

- 외부 노출이 없는 로컬 DB 사용
- 데이터 유실 방지를 위한 자동 SQLite 파일 백업
- 외부 유료 API 미사용으로 인증키·시크릿 관리 대상 없음

## 8. 배포 구성

- 1차: 농장 내 PC에서 `streamlit run app.py`로 로컬 실행
- 2차(선택): Streamlit Community Cloud 배포 검토 — 단, SQLite 파일의 클라우드 휘발성 이슈로 로컬 우선 운영을 기본값으로 유지
- 사용 환경 전제: PC 1대에서 단일 사용자 순차 입력, 모바일 반응형 미지원
